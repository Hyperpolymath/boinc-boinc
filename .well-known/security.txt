# Security Researcher Guidelines
#
# DO:
#   - Report vulnerabilities via security@oblibeny.org
#   - Provide detailed reproduction steps
#   - Allow reasonable time for fixes before disclosure
#   - Encrypt sensitive details using our PGP key
#
# DON'T:
#   - Test against production systems without permission
#   - Access or modify user data
#   - Perform denial of service attacks
#   - Publicly disclose before we've had a chance to fix
#
# In-Scope:
#   - Oblibeny language compiler/interpreter
#   - BOINC server infrastructure
#   - Web dashboard and APIs
#   - Coordinator and validator components
#   - Database access controls
#   - Cryptographic implementations
#
# Out-of-Scope:
#   - Social engineering
#   - Physical attacks
#   - DoS/DDoS attacks
#   - Third-party dependencies (report to them)
#
# Rewards:
#   - Hall of fame recognition
#   - CVE credit
#   - Swag (stickers, t-shirts)
#   - Bounties (when funded): $50-$5000 depending on severity
#
# Safe Harbor:
#   We will not pursue legal action against security researchers who:
#   - Act in good faith
#   - Follow responsible disclosure
#   - Don't violate laws (e.g., CFAA in US)
#   - Avoid privacy violations
#   - Minimize impact to our systems
#
# Thank you for helping keep Oblibeny secure!
